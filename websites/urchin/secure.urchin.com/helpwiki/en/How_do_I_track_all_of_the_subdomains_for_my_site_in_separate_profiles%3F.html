How do I track all of the subdomains for my site in separate profiles?

  • This article authored by ivantage Limited, an Urchin Software Authorized Consultant

Tracking subdomains in individual profiles with Urchin is dependent on the configuration of your webserver. It is possible to configure your web server to record separate log files for each and every sub-domain. Urchin could then process each sub-domain log file seperately. Alternativley, if only one log file is produced and the web access log is set to record the hostname including sub-domain, an include filter could be constructed for every sub-domain's profile. The filter on one profile would include only traffic to, for example, help.example.co.uk and then another include filter would, for example, only include traffic to www.example.co.uk.

So, to track your subdomains in separate profiles, you apply an Include filter to each profile to include only a particular host name. The first step is to create a profile for each subdomain. Once you have done this, you can create the filters described below.

For example, if you would like to track sub.example.com and www.example.com in two separate profiles, apply the following filters:

  • For your sub.example.com profile 

    Filter Type: Include Pattern ONLY
    Filter Field: utm_request_hostname (AUTO)
    Filter Pattern: ^sub\.example\.com
  • For your www.example.com profile 

    Filter Type: Include Pattern ONLY
    Filter Field: utm_request_hostname (AUTO)
    Filter Pattern: ^www\.example\.com
