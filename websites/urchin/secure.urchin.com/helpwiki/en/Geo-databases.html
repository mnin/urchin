Urchin 6.602 Geo-databases

Contributions to this article were made by CODES / webalytics e.K. an Urchin Software Authorized Consultant

The new version of Urchin 6.6.02 now has two Geo-databases installed: the familiar full version as found in Urchin 6.4.00 - 6.6.01 and a new lightweight Geo-database. The reason for the lightweight Geo-database is the high memory utilization of the original version (600-800 MB), which could result in performance problems when processing large log files.

How do I switch between the standard and the lightweight Geo-database?

The primary advantage is a much lower (RAM) memory requirement for processing of log-files. The new lightweight Geo-find database should therefore always be used when speed of protocol processing is important.

To enable or disable the lightweight Geo-database, edit the "etc/urchin.conf" file in the Urchin distribution. The geo.db is used by default. To switch to geo.light.db database, uncomment the "GeodbLight" line in the urchin.conf file: 

######################################################################
# Geo Database Settings
######################################################################
# Uncomment the line below to enable the lightweight geo-database
# GeodbLight: ON

Save the file and restart all Urchin services.

What does the lightweight Geo-database report?

The lightweight Geo-database only reports on the country level and contains no further details in reports. Therefore some of the reports will simply be left blank (not-set).
